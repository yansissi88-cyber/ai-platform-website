## 📋 执行摘要

### 什么是龙虾 (OpenClaw)?
OpenClaw 是 2026 年最火的开源 AI 智能体框架，核心理念是让 AI 从"嘴替"变成"打工人"。用户通过消息软件（微信、Telegram、钉钉等）向"龙虾"发出指令，它就能无间断地自主执行线上任务。

### 为什么 Kira 需要龙虾?
| Kira 现有能力 | 龙虾带来的增强 | 协同价值 |
|--------------|--------------|---------|
| AI Agent Builder | 自主执行脚手架 | 从"配置Agent"到"雇佣数字员工" |
| Campaign 外呼 | 多步骤任务自动化 | 从"单轮呼叫"到"全流程跟进" |
| Workflow 编排 | 动态决策能力 | 从"固定流程"到"智能应变" |
| MCP Store | 工具调用生态 | 从"手动选工具"到"自动选工具" |

### 核心组件

#### 1. 🦞 Lobster Agent (龙虾智能体)
**定义**: Kira 平台上的自主执行代理

**核心能力**:
- **自主规划**: 将用户目标拆解为可执行步骤
- **工具调用**: 通过 MCP 协议调用外部工具
- **记忆持久**: 跨会话保持上下文和状态
- **人机协作**: 关键决策点请求人类确认

**与现有 Agent 的区别**:
| 维度 | 传统 Agent | Lobster Agent |
|-----|-----------|--------------|
| 交互模式 | 一问一答 | 自主执行多步骤 |
| 任务长度 | 单轮/短对话 | 长期任务(数小时/天) |
| 错误处理 | 立即报错 | 自动重试/迂回 |
| 人工介入 | 需主动查询 | 主动推送关键决策 |

#### 2. 🧠 Model Router (智能模型路由)
**策略**: 根据任务类型、成本预算、成功率要求自动选择最优模型

**路由规则示例**:
```javascript
{
  "coding_task": {
    "high_stakes": "gemini-3-flash",  // 成功率95.1%
    "balanced": "minimax-m2.1",        // 性价比最优
    "fast": "minimax-m2.5"             // 速度最快
  },
  "customer_service": {
    "default": "kimi-k2.5",            // 长文本理解
    "complex": "claude-sonnet-4.5"     // 复杂情感处理
  }
}
```

#### 3. 🛡️ Secure Sandbox (安全沙箱)
**多层安全机制**:
- **租户隔离**: 每个企业独立虾房，数据完全隔离
- **权限控制**: 细粒度 ACL，控制虾可访问的工具和数据
- **审计日志**: 完整记录虾的所有操作
- **人工审核**: 高风险操作触发审批流

## 🎨 产品功能设计

### 功能 1: 虾房管理器 (Lobster Pen)
**定位**: 企业管理员配置和管理龙虾员工的控制台

**核心功能**:
1. **虾的创建与配置**
   - 选择"虾种"(模型类型: MiniMax/Kimi/Claude等)
   - 配置虾的"技能包"(MCP工具集)
   - 设定虾的"工作范围"(可访问的系统和数据)

2. **虾群编排**
   - 多虾协作: 销售虾 + 客服虾 + 数据分析虾
   - 任务分配: 基于负载和能力自动分配
   - 交接机制: 复杂任务多虾接力

3. **监控与审计**
   - 实时看板: 每只虾的工作状态
   - 成本追踪: Token消耗、任务成本
   - 操作日志: 完整审计追踪

**UI 原型**:
```
┌────────────────────────────────────────────┐
│ 🦞 虾房管理器          [创建新虾] [批量操作] │
├────────────────────────────────────────────┤

│ 🦞 销售虾-001    🦞 客服虾-002    🦞 数据虾-003 │
│ ● 工作中          ● 空闲           ● 休眠   │
│ 模型: MiniMax    模型: Kimi      模型: Claude│

│ 成功率: 94%      成功率: 96%     成功率: 97% │
│                                            │
│ [查看详情] [暂停] [查看详情] [唤醒] [查看详情]│
│                                            │
└────────────────────────────────────────────┘
```

### 功能 2: 对话式任务中心 (ChatOps)
**定位**: 员工与龙虾协作的统一入口

**交互模式**:
- **自然语言指令**: "帮我跟进昨天未接通的客户，重新安排外呼时间"
- **富媒体反馈**: 虾返回进度卡片、数据图表、确认按钮
- **人机混合**: 虾处理繁琐工作，人类处理关键决策

**典型工作流程**:
```
用户: "帮我准备一个针对金融行业客户的产品介绍方案"

虾: "收到！我来为您准备方案。需要确认几个信息：
    1. 目标客户规模？(大型/中型/小型)
    2. 重点突出哪些功能？
    3. 方案格式？(PPT/Word/网页)"

用户: "大型客户，重点突出安全合规和私有化部署，PPT格式"

虾: "正在生成方案...[进度条]
    ✓ 已检索金融行业案例库
    ✓ 已提取安全合规认证信息
    ✓ 正在生成PPT...[预计2分钟]"

[2分钟后]

虾: "方案已生成！
    📎 《金融行业大客户解决方案.pptx》
    
    包含内容：
    - 行业痛点分析 (3页)
    - Kira安全合规架构 (5页)
    - 私有化部署方案 (4页)
    - 同行业案例 (2页)
    
    需要我调整哪些部分？"
```

### 功能 3: 智能外呼进化版 (Campaign Lobster)
**定位**: Campaign 功能的龙虾增强版

**核心增强**:
1. **多轮自主跟进**
   - 传统: 单轮呼叫，挂断即结束
   - 龙虾: 分析挂断原因，自动安排再呼叫时间

2. **动态脚本生成**
   - 基于客户画像实时生成个性化话术
   - 根据对话进展调整策略

3. **跨渠道协调**
   - 电话未接通 → 自动发邮件/短信
   - 客户回复邮件 → 虾阅读并安排后续

**示例场景**:
```javascript
// 龙虾执行的外呼任务
{
  "task": "金融产品推广",
  "contact": "张总 (某银行CIO)",
  "history": [
    { "time": "昨天14:00", "result": "未接通", "action": "安排今早9:30再试" },
    { "time": "今天09:30", "result": "接通，表示忙", "action": "询问方便时间，预约明天下班后" },
    { "time": "今天09:35", "action": "发送产品简介邮件", "status": "已发送" }
  ],
  "next_action": "明天18:00准时回拨",
  "preparation": "准备《银行数字化转型白皮书》"
}
```

### 功能 4: 自进化知识库 (Auto-KB)
**定位**: 让龙虾在工作中持续学习和优化

**工作机制**:
1. **成功模式提取**: 虾完成任务后，提炼成功策略
2. **失败原因分析**: 失败任务自动归因，生成改进建议
3. **知识沉淀**: 将经验转化为可复用的"虾谱"(Prompt Template)

**知识库类型**:
- **行业虾谱**: 金融/医疗/教育等垂直领域最佳实践
- **场景虾谱**: 售前/售后/续费等场景话术库
- **客户虾谱**: 特定客户的偏好和历史交互模式

```
       ┌──────────────────┼──────────────────┐
       │                  │                  │

└─────────────┘  └─────────────────┘  └────────────┘
```

#### 1. Agent Builder + Lobster = Auto-Agent
- 可视化配置 Agent 时，可勾选"启用自主执行模式"
- 传统 Agent 成为 Lobster 的"大脑"，负责决策
- Lobster 负责"手脚"，负责执行

#### 2. Campaign + Lobster = Smart Campaign
- Campaign 创建时，选择"龙虾模式"
- 虾自动分析客户列表，制定个性化外呼策略
- 实时调整外呼时间和话术

#### 3. Workflow + Lobster = Dynamic Workflow
- Workflow 节点可配置"龙虾决策点"
- 在复杂分支时，让虾根据上下文动态选择路径
- 支持"人工确认"节点，虾暂停等待人类输入

#### 4. Inbox + Lobster = 智能收件箱
- 虾自动阅读客户邮件，生成摘要
- 自动起草回复，供人工审核后发送
- 复杂问题自动转人工，简单问题自动处理

### 定价策略

#### 方案 A: 按虾租赁 (Lobster as a Service)
| 虾种 | 月租 | 包含Token | 适用场景 |
|-----|-----|----------|---------|
| 🦐 小虾 (MiniMax M2.5) | ¥99/月 | 1M tokens | 个人/小团队 |
| 🦞 中虾 (Kimi K2.5) | ¥199/月 | 2M tokens | 中小企业 |
| 🦞 大虾 (Claude Sonnet) | ¥499/月 | 1M tokens | 复杂任务 |
| 🦞🦞 虾群 (多虾协作) | ¥999/月起 | 按需 | 企业级 |

**超额计费**: Token用尽后，按实际使用量计费

#### 方案 B: 按任务付费 (Pay per Task)
- 简单任务 (FAQ回复): ¥0.5/次
- 标准任务 (客户跟进): ¥2/次
- 复杂任务 (方案生成): ¥10/次
- 企业客户包月不限量: ¥5000/月起

#### 方案 C: 混合模式 (推荐)
- 基础月租 + 超额任务费
- 例: ¥199/月(含100次标准任务) + ¥1.5/次超额

### 成本结构分析
**以"中虾"(MiniMax M2.1)为例**:
- 模型成本: ~$0.14/任务 (约¥1)
- 平台运维: ~¥0.3/任务
- **毛利**: 定价¥2 - 成本¥1.3 = **35%毛利率**
